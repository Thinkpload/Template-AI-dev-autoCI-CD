<task type="auto" tdd="false">
  <name>Task 1: Add actor guard and Security Audit step to ci.yml</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Make two targeted edits to `.github/workflows/ci.yml`:

    **Edit 1 — Job-level actor guard (CI-04).**
    After the `runs-on: ubuntu-latest` line (line 12), add:
    ```yaml
        if: github.actor != 'github-actions[bot]'
    ```
    This prevents the `github-actions[bot]` user from triggering the build job at all — belt-and-suspenders alongside GITHUB_TOKEN's built-in loop prevention.

    **Edit 2 — Security Audit step (CI-01).**
    After the "Install dependencies" step and before the "Check no @latest in wizard source" step, insert:
    ```yaml
          - name: Security Audit
            run: npm audit --audit-level=high > audit.log 2>&1 || (cat audit.log && exit 1)
    ```
    Key behavior: exits non-zero ONLY when a high or critical vulnerability is found; exits 0 for low/moderate. The `|| (cat audit.log && exit 1)` pattern prints the log before failing so the error is visible in the CI run output.
  </action>
  <verify>
    <automated>grep -n "github.actor != 'github-actions\[bot\]'" .github/workflows/ci.yml && grep -n "npm audit --audit-level=high" .github/workflows/ci.yml</automated>
  </verify>
  <done>Both lines are present in ci.yml. Actor guard appears under the `build:` job block before the `steps:` key. Audit step appears between "Install dependencies" and "Check no @latest".</done>
</task>
