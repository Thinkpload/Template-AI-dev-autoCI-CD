<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create GitHub repository labels</name>
  <action>
    This is a Wave 0 prerequisite requiring human execution. Run these three gh CLI commands in your terminal from the repo root (requires `gh auth login` completed):

    ```bash
    gh label create "bug" --color "d73a4a" --description "Something is not working" --force
    gh label create "ci-failure" --color "e4e669" --description "CI pipeline failed" --force
    gh label create "needs-human" --color "e11d48" --description "Auto-fix exhausted — requires human review" --force
    ```

    The `--force` flag updates the label if it already exists, making this safe to re-run.

    Why this cannot be automated: `gh label create` requires an authenticated GitHub session tied to the actual repository. Claude cannot run these commands against the live GitHub API without the user's active `gh auth` session.
  </action>
  <verify>
    <automated>gh label list | grep -E "bug|ci-failure|needs-human"</automated>
  </verify>
