## Integration Findings

### Critical Breaks

**Break 1 — No root package.json (affects CI-01, CI-02, QA-04)**

`ci.yml` runs all commands at the repository root. There is no `package.json` at root — only `wizard/package.json` exists. `npm ci` fails immediately on every CI run with `npm error enoent Missing package.json`. All downstream steps (Security Audit, @latest grep, Lint, Tests, SonarCloud, Build) are unreachable.

The "Report CI Failure" step does fire (it has `if: failure()`) and creates a GitHub Issue — but the issue body captures an npm installation failure, not an actionable CI failure.

Fix required:
```yaml
# In ci.yml build job:
defaults:
  run:
    working-directory: wizard

# In setup-node step:
cache-dependency-path: wizard/package-lock.json
```

**Break 2 — GSD activation not implemented (affects AI-02)**

`MODULE_REGISTRY.gsd` has `devDeps: []` and `postInstall: []`. A registry comment says "Phase 3 handles GSD activation logic" but Phase 3 never fulfilled this. Selecting GSD in the wizard copies only a `.gitkeep` placeholder. No npm packages are installed, no configuration is written.

**Break 3 — sonar-project.properties path mismatch (affects QA-04 in CI context)**

Root `sonar-project.properties` has `sonar.sources=src` (no `src/` exists at root; wizard source is at `wizard/src/`) and `sonar.javascript.lcov.reportPaths=coverage/lcov.info` (actual coverage path is `wizard/coverage/lcov.info`). SonarCloud would scan a nonexistent directory and upload no coverage data.

Note: The wizard correctly installs a properly-configured `sonar-project.properties` into target projects via `wizard/templates/vitest/sonar-project.properties`. The misconfiguration is in the template repo's own CI scan, not the output it generates.

**Break 4 — fix-issue.md uses npm test without working-directory (affects Flow 4)**

`fix-issue.md` Step 4 runs `npm test` without specifying `cd wizard` first. Any fix PR produced by `/fix-issue` would still fail CI for the same reason as Break 1 — `npm test` at root has no `package.json`.
